# 24 周毕业研究报告模板

> 建议正文控制在 20–40 页，技术细节、完整参数和附加图表放入附录。每张核心表图必须标注实验 ID、数据版本、样本区间及成本口径。删除本提示后提交。

## 封面

- 项目标题：
- 作者 / 复核人：
- 提交日期：
- 报告版本：
- 最终实验 ID：
- 代码版本：
- 数据快照版本：
- 结论：`继续 / 重做 / 停止`

## 摘要

用 300–500 字回答：

1. 研究了什么可证伪问题？
2. 经济机制是什么？
3. 使用什么数据、样本外设计和基线？
4. 成本后主要结果及 95% 置信区间是什么？
5. 最重要的风险、限制和最终决策是什么？

不得在摘要只报告最佳区间、成本前结果或样本内结果。

## 1. 研究问题与预注册

### 1.1 决策问题

- 策略或信号服务于什么投资决策：
- 目标持有期、调仓频率与适用资金规模：
- 不在本报告范围内的问题：

### 1.2 机制与假设

- 信息、行为或结构性机制：
- 主要假设与预期方向：
- 零假设：
- 至少三项可证伪条件：
- 可能的替代解释：

### 1.3 预注册与偏离

- 研究日志 ID：
- 预注册日期：
- 主要指标与通过阈值：
- 候选尝试预算：
- 最终测试冻结规则：
- 与预注册的全部偏离及其影响：

## 2. 数据与样本

### 2.1 数据来源

逐项列出供应商、数据集 ID、快照时间、内容哈希、模式版本、许可证限制和已知缺陷。

### 2.2 时间点正确性

- 事件时间、公开时间、系统可知时间与最早交易时间：
- 财务公告和修订处理：
- 公司行动与复权处理：
- 宏观数据初值与修订值：
- 随机时点审计样本数、方法和违规数：

### 2.3 历史股票池

- 股票池定义及版本：
- 上市等待期、退市、ST、停牌和流动性规则：
- 历史指数成分与行业分类处理：
- 各排除原因的样本数量：

### 2.4 数据质量

- 主键重复、关键字段缺失和异常值：
- 交易日历、OHLC、复权和公司行动交叉检查：
- 覆盖率与缺失率随时间变化：
- 数据漂移与供应商变更：
- 未解决问题及影响：

## 3. 方法

### 3.1 信号或模型

- 公式、特征、方向与参数：
- 回看窗口和最小历史长度：
- 去极值、缺失填充、中性化与标准化顺序：
- 预处理拟合范围：
- 模型选择和超参数：
- 随机种子与非确定性：

### 3.2 标签与基准

- 标签定义、起止时间和基准：
- 重叠标签处理：
- 简单策略基线：
- 选择该基线的理由：

### 3.3 验证设计

- 训练、验证、外层测试和最终测试区间：
- walk-forward、purge 与 embargo：
- 特征和超参数选择范围：
- 有效候选尝试总数：
- 多重检验和选择偏差修正：
- bootstrap 或置信区间方法：

### 3.4 组合构建

- 信号到权重的映射：
- 单股、行业、风格、换手和流动性约束：
- 风险模型与协方差估计：
- 优化目标、求解器和不可行时降级：
- 目标权重到可执行权重的转换：

### 3.5 成交模型

- 信号可用时间与执行延迟：
- 成交价格假设：
- 佣金、最低佣金、税和价差：
- 冲击成本与参与率模型：
- 停牌、涨跌停、T+1、整数手和部分成交：
- 未成交订单处理：

## 4. 结果

### 4.1 样本与覆盖

报告证券数、证券—日数、调仓数、持仓数、覆盖率、拒单率和未成交率。

### 4.2 预测或因子效果

- Rank IC、ICIR 及 95% 置信区间：
- 五分组收益与单调性：
- 分年度、行业、市值和市场状态结果：
- 因子相关与边际贡献：
- 模型相对简单基线的增量：

### 4.3 组合绩效

必须同时报告策略、基线和基准：

- 累计与年化收益：
- 年化波动、Sharpe、信息比率和 Calmar：
- 最大回撤、恢复期、95% VaR 与 ES：
- 月度胜率、年度收益和最差月份：
- 换手、集中度和现金比例：
- 成本前后差异：

### 4.4 风险与归因

- 行业和风格主动暴露：
- 因子风险与特异风险：
- 因子、选股、成本和残差收益贡献：
- 归因对账误差：
- 预测风险与实现风险偏差：

### 4.5 容量

- 规模、参与率、冲击成本与净收益曲线：
- 正常、流动性减半与成本翻倍情景：
- 容量上限定义和估计值：
- 高换手日和未成交订单影响：

## 5. 稳健性与反证

逐项给出设计、结果与结论，不可只写“通过”：

- 参数和窗口扰动。
- 样本起止时间扰动。
- 股票池、行业分类和基准替换。
- 成本翻倍、延迟增加、流动性减半。
- 市场状态与历史压力窗口。
- 安慰剂、随机信号或事件日平移。
- 删除核心特征或替代数据源。
- 样本协方差与收缩协方差。
- 嵌套验证与普通切分对比。

明确列出最不利结果，以及它是否推翻主要假设。

- 七类故障演练结果：
- 持仓、现金、订单和成交对账：

## 8. 局限、适用边界与伦理

至少说明：

1. 数据覆盖、准确性和供应商依赖。
2. 统计功效、非平稳和模型不确定性。
3. 交易成本、容量和市场冲击估计误差。
4. 极端市场、停牌和基础设施风险。
5. 拥挤交易、市场影响和策略失效。
6. 本研究不能支持的因果或收益承诺。

列出至少三项会推翻当前结论的新证据。

## 9. 结论与决策

- 最终决策：`继续 / 重做 / 停止`
- 主要证据：
- 反对证据：
- 证据强度：
- 策略适用边界：
- 风险预算和容量上限：
- 下一项最小必要验证：
- 禁止对外声称的内容：

即使策略被停止，只要研究问题明确、流程正确、证据透明且可复现，也可构成合格毕业成果。

## 附录 D：毕业验收检查表

### 研究完整性

- [ ] 单一主要问题、机制、零假设和可证伪条件清楚。
- [ ] 主要指标、阈值和最终测试规则已预注册。
- [ ] 全部候选、失败实验和人工干预已披露。

### 数据与时点

- [ ] 原始快照、内容哈希、模式版本和数据血缘齐全。
- [ ] 历史股票池、退市、财务公告与修订处理正确。
- [ ] 100 条随机时点审计无未来信息违规。

### 验证与绩效

- [ ] purge、embargo、嵌套验证和最终测试隔离正确。
- [ ] 样本外、成本后结果优先，并与简单基线比较。
- [ ] 报告置信区间、多重检验和真实尝试次数。
- [ ] 风险归因可对账，稳健性和压力测试完整。

### 执行与治理

- [ ] 成本、冲击、部分成交和容量模型有依据。
- [ ] 约束求解、目标与实际持仓偏差有记录。
- [ ] 监控、告警、降级、回滚和故障演练通过。

### 复现

- [ ] 所有核心表图可追溯到实验 ID。
- [ ] 最终环境、代码、数据和配置不可变且可获得。
- [ ] 独立复现达到预设数值容差。
- [ ] 必需产物和检查表完成率 100%。

## 附录 E：答辩问题

1. 哪一项结果最可能是选择偏差造成的？
2. 若成本翻倍或容量扩大四倍，结论如何变化？
3. 为什么该基线足够有竞争力？
4. 哪个数据字段最可能引入未来函数？
5. 哪项反证最接近推翻核心假设？
6. 模型在哪种市场状态下最脆弱？
7. 若明天数据源延迟或模型失败，系统会做什么？
8. 为什么最终决策是继续、重做或停止？
