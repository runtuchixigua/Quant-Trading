# 第 1 周：收益与风险

## 学习目标

学完本周，你应能从价格序列独立计算简单收益率、复利净值、回撤和最大回撤；能区分“收益有多高”与“实现收益的路径有多危险”；能解释年化收益、年化波动、Sharpe 和 Calmar 各自回答什么问题；还能从 CSV 明细反查汇总指标，而不是把指标函数当成黑箱。

本实验使用固定随机种子的离线合成行情。它适合验证计算链路和训练解释能力，但不能证明任何真实市场规律，也不代表未来收益。

## 前置知识

需要掌握百分数、小数、乘法与平方根，理解 pandas 表格中的“行、列、索引”。约定收益率以小数存储，例如 `0.02` 表示 2%，`-0.01` 表示 -1%。所有计算按交易日顺序进行，第一天因没有前一日价格，程序将收益填为 0。

## 核心理论与公式推导

设第 \(t\) 日收盘价为 \(P_t\)，简单收益率为：

\[
r_t=\frac{P_t}{P_{t-1}}-1
\]

由此可反推 \(P_t=P_{t-1}(1+r_t)\)。若初始净值 \(W_0=1\)，每日把全部财富继续投入，则：

\[
W_t=W_{t-1}(1+r_t)=\prod_{i=1}^{t}(1+r_i)
\]

总收益不是日收益之和，而是：

\[
R_{\mathrm{total}}=W_T-1
\]

运行峰值是截至当日出现过的最高净值：

\[
H_t=\max_{0\le i\le t}W_i
\]

当前回撤定义为：

\[
D_t=\frac{W_t}{H_t}-1
\]

因此回撤通常不大于 0；创新高时 \(W_t=H_t\)，回撤为 0。最大回撤是 \(\min_t D_t\)，本项目的 `max_drawdown` 保留负号。最大回撤必须从路径中计算，不能只由起点和终点决定。

若一年按 252 个交易日，样本含 \(N\) 个日收益，则常见年化形式为：

\[
R_{\mathrm{ann}}=W_T^{252/N}-1,\qquad
\sigma_{\mathrm{ann}}=\operatorname{std}(r_t)\sqrt{252}
\]

忽略无风险利率时，Sharpe 近似为年化收益相对年化波动的比值；Calmar 近似为年化收益除以最大回撤绝对值。二者分别强调波动风险和深度损失风险，不能互相替代。

## 用生成数据行手算

取讲义中的三次价格变化作为演示，不要求与完整 CSV 的具体价格相同：

| 日 | close | return | wealth | running peak | drawdown |
|---|---:|---:|---:|---:|---:|
| 0 | 100.00 | 0 | 1.000000 | 1.000000 | 0 |
| 1 | 102.00 | \(102/100-1=0.02\) | \(1\times1.02=1.020000\) | 1.020000 | 0 |
| 2 | 99.96 | \(99.96/102-1=-0.02\) | \(1.02\times0.98=0.999600\) | 1.020000 | \(0.9996/1.02-1=-0.02\) |
| 3 | 104.958 | \(104.958/99.96-1=0.05\) | \(0.9996\times1.05=1.04958\) | 1.049580 | 0 |

这个例子揭示两个重点。第一，+2% 后 -2% 并未回到原点，因为 \(1.02\times0.98=0.9996\)。第二，第 2 日相对前一日和历史峰值都下降 2%，所以当日回撤是 -2%；第 3 日创新高后回撤归零。若把同样的日收益重新排序，最终乘积不变，但中途峰值与谷底的位置可能改变，最大回撤也可能改变。

## 实验步骤

1. 运行第 1 周课程。普通模式生成 252 个交易日，快速模式生成 60 个交易日；两者均为单资产、固定 seed=1 的合成行情。
2. 程序按日期索引取得 `close`，用 `pct_change(fill_method=None)` 计算简单收益，并仅把第一项缺失值填为 0。
3. 对收益逐日复利得到 `wealth`，再以运行峰值计算 `drawdown`。
4. 用同一收益序列生成全期绩效汇总，保存为 `metrics.csv`。
5. 绘制上下排列且共享日期轴的净值和回撤图。
6. 不先查看实现，另写一段 NumPy 或 pandas 代码复算，再逐行比较误差。

## 按文件阅读顺序与逐字段解释

### 1. `daily_data.csv`

这是事实底稿，应最先阅读。

- `date`：交易日，也是 CSV 写出时的索引标签。检查日期应单调递增且无重复。
- `close`：当日合成收盘价。它是计算收益的输入，不是策略预测。
- `return`：相邻收盘价的简单收益率。首行固定为 0；第二行起应满足 `close / close.shift(1) - 1`。
- `wealth`：从 1 起步的复利财富指数。任意一行都应满足前一行净值乘以 \(1+\text{return}\)。
- `drawdown`：当前净值相对历史最高净值的比例差。范围应不高于 0，创新高日等于 0。

抽查时至少验证首行、一个上涨日、一个下跌日和最大回撤日。浮点数比较应设置容差，不要要求十进制文本完全一致。

### 2. `metrics.csv`

第一列在文件中没有显式列名，它实际保存指标名称；本 frontmatter 用 `unnamed_index` 表示该索引列。第二列 `value` 保存指标值。行指标包括：

- `total_return`：期末 `wealth - 1`。
- `annualized_return`：按样本交易日数把复利结果折算到一年。
- `annualized_volatility`：日收益标准差乘 \(\sqrt{252}\)。
- `sharpe`：单位波动对应的超额收益近似值；负值表示风险调整后表现不佳。
- `max_drawdown`：最深回撤，项目中为负数。
- `calmar`：年化收益相对最大回撤绝对值的比率。

先由明细复算，再读汇总；若两者不一致，应优先检查年化口径、标准差自由度、首日 0 收益和最大回撤符号。

### 3. `wealth_and_drawdown.png`

上图纵轴是 `wealth`，1.10 表示初始财富增长约 10%，不是“当日收益 110%”。下图纵轴是 `drawdown`，越向下表示距离历史峰值越远。重点识别净值峰值、最大回撤谷底和是否恢复到前高。

### 4. `homework.md`

它要求独立复算、定位峰值日/谷底日/恢复日、打乱收益顺序 100 次，并解释算术平均收益为何不能代表长期复利。答案应包含代码、数字和解释。

### 5. `acceptance.json`

它保存验收条件，而非成绩结果。应确认你提交了自己的计算代码，并能解释“总收益相同而最大回撤不同”的路径依赖。

## 图表解读

先看上图整体方向，再看下图风险过程。净值长期上涨但回撤频繁，说明收益并非平滑获得；净值终点接近 1 但回撤很深，说明投资者曾承受显著中途损失；长时间负回撤代表长期未创新高。不能仅凭曲线“好看”判断策略有效，还需要样本外检验、基准、成本和统计可信度。

## 动态指标判断

每增加一个交易日，都重新检查：当日收益是否异常、净值是否创新高、当前回撤是否继续扩大、滚动波动是否突升。若 `wealth` 与累计乘积不一致、`drawdown` 大于 0、价格非正或日期乱序，应先判为数据/计算故障。风险判断不能只等全期结束：回撤接近预先设定阈值时应降风险或暂停调查；新高不意味着风险消失，仍需观察波动和收益集中度。

## 常见误区

1. 把日收益直接求和当总收益，忽略复利。
2. 把平均日收益乘 252 当作严格年化复利收益。
3. 把 `drawdown=-0.20` 读成正收益 20%，或把最大回撤的负号删掉后与其他实现混比。
4. 只看期末净值，忽略中途可能无法承受的损失路径。
5. 把合成数据上的指标当作真实 A 股证据。
6. 用未来峰值计算当前回撤；正确峰值只能使用截至当日的数据。

## 思考题

1. 为什么 +50% 后 -50% 的总收益不是 0？
2. 两条路径最终净值相同，哪一条的最大回撤一定更小吗？需要什么额外信息？
3. 收益序列打乱后，哪些统计量不变，哪些会变？
4. Sharpe 相同的两个策略，最大回撤和恢复时间可能为何不同？
5. 若首日收益保留为缺失值而非 0，年化样本数应如何处理？

## 作业提示

使用 `cumprod(1 + return)` 复算净值；使用 `cummax` 得到运行峰值；最大回撤谷底是 `drawdown.idxmin()`。峰值日应在谷底日之前，从谷底向后寻找净值首次达到旧峰值的日期作为恢复日。打乱实验每次保存总收益与最大回撤，并报告分布，而不是只展示一个随机结果。
