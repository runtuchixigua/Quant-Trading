# 第 4 周：交易成本与成交约束

## 学习目标

本周要把“毛收益”变成“可实现净收益”。你应能区分佣金、滑点和印花税的计费对象；能解释换手为何把微小费率放大；能设计并完整报告成本敏感性网格；能判断策略是否只在乐观成本假设下成立。实验以 20/60 均线策略为固定对象，不借调整策略参数来掩盖成本影响。

## 前置知识

需要理解第 3 周的目标权重、成交权重、逐日收益和换手。费率均以小数表示，例如 `0.0003` 是成交额的 3 个基点，即 0.03%。印花税在本实验中只对卖出成交额计收；佣金和滑点作用于双边成交。

## 核心理论与公式

设第 \(t\) 日买入成交额为 \(B_t\)，卖出成交额为 \(S_t\)，佣金率 \(c\)，滑点率 \(s\)，卖出印花税率 \(\tau\)，教学模型中的成本可写为：

\[
C_t=c(B_t+S_t)+s(B_t+S_t)+\tau S_t
\]

净收益近似为：

\[
r^{net}_t=r^{gross}_t-\frac{C_t}{NAV_{t-1}}
\]

若以权重变化近似双边换手：

\[
\mathrm{turnover}_t\approx\sum_i|w^{exec}_{i,t}-w^{held}_{i,t-1}|
\]

则成本拖累大致与费率乘以换手成正比。高换手策略即使单次费率很低，长期累计成本也可能吞噬全部毛收益。滑点不是固定行政收费，而是成交价相对参考价的不利偏移；真实滑点通常随流动性、订单方向、订单规模和市场状态变化，本周仅用固定率建立最小可审计链路。

网格使用三类成本各三档：0、基准、2 倍。基准为佣金 0.0003、滑点 0.0005、印花税 0.0005，共：

\[
3\times3\times3=27
\]

组。预先确定并完整保留所有结果，可减少只挑有利假设的选择偏差。

## 实验步骤

1. 用 seed=404 生成合成行情并固定 20/60 均线目标权重。
2. 分别把三类成本设为 `zero`、`base`、`double`，形成 27 个配置。
3. 每个配置使用相同价格、信号和执行时间线运行回测。
4. 记录实际费率、累计成本、总收益、年化收益、Sharpe 和最大回撤。
5. 按实际费率排序保存，不删除负收益或不利组合。
6. 对每个佣金—滑点组合，在三个印花税档位上计算平均总收益并画图。
7. 分别按一种成本分组，在其余成本上取平均，估计边际影响。

## 按文件阅读顺序与逐字段解释

### 1. `cost_grid.csv`

- `commission_level`：佣金档位标签 `zero/base/double`。
- `slippage_level`：滑点档位标签。
- `stamp_duty_level`：印花税档位标签。
- `commission`：实际佣金率，基准为 0.0003。
- `slippage`：实际滑点率，基准为 0.0005。
- `stamp_duty`：实际卖出印花税率，基准为 0.0005。
- `total_cost`：全期实际扣除的成本总额或归一化成本累计值，按回测实现口径解释。
- `total_return`：扣除该配置成本后的全期复利收益。
- `annualized_return`：按样本长度折算的净年化收益。
- `sharpe`：净日收益的年化 Sharpe。
- `max_drawdown`：净值相对历史峰值的最深回撤，通常为负。

首先确认恰有 27 行且三档笛卡尔积完整，再比较只改变一项费率的配对行。不能把 level 标签当数值排序依据，应使用实际费率列。

### 2. `cost_analysis.md`

该报告给出网格规模、最高和最低总收益，并强调所有结果均保留。它还说明印花税仅作用于卖出，佣金和滑点作用于双边换手，以及合成数据只能验证成本传导，不能证明策略有效。

### 3. `cost_sensitivity.png`

横轴是佣金档位，每组柱的颜色代表滑点档位；纵轴是跨三个印花税档位平均后的总收益。图中没有单独呈现印花税差异，因此不能凭该图判断印花税边际影响，必须回到 CSV。

### 4. `homework.md`

需要计算三种成本各自从零到基准、从基准到双倍的平均变化，找出成本翻倍后结论反转的配置，并联系换手解释敏感性。

### 5. `acceptance.json`

验收关注网格是否完整、三类成本是否解释正确，以及能否基于合理成本范围判断策略脆弱性。

## 图表解读

若佣金或滑点升档时柱形整体下降，说明成本传导方向合理。下降幅度近似线性只反映固定费率模型；真实冲击成本可能非线性。若图中高成本反而收益更高，应检查执行或成本符号。由于图对印花税取平均，任何关于单边税负的结论都要用 CSV 配对验证。

## 动态指标判断

实盘或模拟盘应每日监控成交额、双边换手、卖出成交额、各类成本、单位换手成本和净/毛收益差。单位成本突然抬升可能来自流动性恶化、订单规模变大或模型错误。策略滚动毛收益尚可但净收益转负时，应暂停扩张；若成本占毛收益比例长期过高，优先降低换手或改善执行，而不是继续微调预测参数。

## 常见误区

1. 只跑零成本或单一乐观成本。
2. 把印花税错误地收在买卖双边。
3. 把固定滑点当成真实市场冲击的完整模型。
4. 使用目标换手而非实际成交换手计费。
5. 只报告最优网格，隐藏策略在合理成本下失效。
6. 比较网格时同时改变信号或样本，导致无法归因。
7. 把总成本金额与收益率直接相减而忽略净值尺度。

## 思考题

1. 两个总换手相同的策略，真实滑点为何可能不同？
2. 卖出受限时，当期成本下降是否意味着结果更好？
3. 固定费率模型在哪些市场状态下会严重低估成本？
4. 若最低佣金存在，如何修改小额订单成本公式？
5. 怎样把成交参与率和策略容量加入敏感性分析？

## 作业提示

使用配对差分：固定另外两个档位，仅改变一个成本档位；再对所有配对求均值和分布。定义“结论反转”前先声明阈值，例如总收益从正变负、Sharpe 从正变负，或相对基准超额消失。报告中同时列绝对变化和基点变化。

## 验收标准

- `cost_grid.csv` 有且仅有 27 个唯一成本组合。
- 三个费率数值与 `zero/base/double` 标签一致。
- 能解释佣金、滑点和印花税各自作用的成交额。
- 能复算至少一行 `total_cost` 的构成。
- 能量化三类成本的边际影响，不只看图。
- 能明确判断策略在合理成本区间内是稳健还是脆弱，并说明依据。
